Git

Git - populiariausia kodo versijavimo programa.

Git komandos

  • git init - sukuria git repozitoriją dabartiniame aplankale
  • git status - nurodo informaciją apie lokalios git repozitorijos statusą
  • git add - prideda nurodytus failus prie commit paruoštų failų (stage)
  • rm --cached - išima nurodytus failu iš stage zonos (unstage)
  • commit -m "žinutė" - sukuria kodo versiją iš stage failų ir prideda žinutę apie pakitimus (ne ilgesnė nei 50 simbolių)
  • restore - grąžina nurodytus failus į buvusio commit būseną (panaikina naujus pakitimus po paskutinio commit)
  • push - iš lokalios repozitorijos nusiunčia commit į išorinę repozitoriją
  • pull - iš išorinės repozitorijos atsiunčia commit į lokalią repozitoriją
  • clone - klonuoja nurodytą repozitoriją iš išorinės repozitorijos
  • log - išveda commit sąrašą
  • chechout commitID|branchName - pareina į nurodytą kodo versiją ARBA
  • branch naujosSakosPavadinimas - sukuria naują šaką

Branch protection rules

Vienas svarbiausių šakų apsaugos taisyklių yra "Pull Request" (bandymas vienos šakos commit sukelti į kitą šaką).
Pull Request dar galima suteikti "require approvals", kuris reikalauja, kad nurodytas kiekis kitų projekto dalyvių patvirtintų Pull Request.

Eiliškumas Pull Request

  1. Susikurti naują šaką - git branch newBranchName
  2. Nueiti į tą naują šaką - git checkout newBranchName
  3. Atlieki užduotį, kurią apsiėmei padaryti
    1. Add pakitusius failus - git add .
    2. Commit pakitimus - git commit -m "Žinutė apie pakitimus"
  4. Nueini į main/master šaką - git checkout main|master
  5. Atnaujini main/master šaką - git pull
  6. Nueini į tą naują šaką - git checkout newBranchName
  7. Sujungi savo šaką su main ir išsprendi galimai atsiradusius konfliktus - git merge main/master
  8. Musiunti tos naujos šakos pakitimus į GitHub - git push
  9. GitHub sukurti Pull Request
  10. Kiti komandos nariai patvirtina arba atmeta Pull Request
